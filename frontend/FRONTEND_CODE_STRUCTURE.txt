Current frontend architecture summary - 2026-05-13
--------------------------------------------------

The frontend uses a feature-first React structure:
- Route definitions live in `src/app/router.jsx`.
- Global cross-cutting state lives in `src/context`.
- Feature modules own their pages, components, services, query hooks, and realtime hooks.
- Shared API behavior is centralized in `src/shared/api`.
- Shared visual language is centralized in `src/index.css`, `src/shared/ui`, `src/shared/components`, and `src/layouts`.
- Realtime subscriptions are isolated in feature-specific `realtime/` folders plus the shared SignalR client.
- Auth context carries backend account-state flags so UI can show read-only confirmation/suspension affordances while the backend remains authoritative.
- Production deployment is handled by `npm run build:backend`, which copies the compiled app into the backend host.

Current UI implementation summary - 2026-05-11
----------------------------------------------

- Normal user surfaces now use shared page headers, controls, messages, loading states, empty states, cards, tabs, and dialogs across media, account, watchlist, groups, and collection detail.
- Admin/media workflows now use the shared system for media management, add/edit media, media cart, season management, and media status controls.
- Moderator/admin operational workflows now use shared filters, badges, dialogs, inline messages, empty/loading states, and action rows in moderation reports, assignments, audit logs, group moderation actions, and collection/group destructive actions.
- New work should prefer shared primitives and `ui-*` classes before adding local button, field, card, dialog, table, or message styles.
