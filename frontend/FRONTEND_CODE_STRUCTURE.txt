Architecture audit notes - 2026-04-26
-------------------------------------

Current frontend structure is feature-based React + Vite:
- src/app owns provider composition and the active router in src/app/router.jsx, rendered by src/app/AppRouter.jsx.
- src/context owns global auth/theme/language/media-cart state.
- src/features owns page/components/query/service modules by domain.
- src/shared owns API clients, layout primitives, shared UI pieces, signalR, constants, and utility helpers.

Important quality issues to address:
- The global SearchBar currently logs to console instead of navigating/searching. Make it route to /media?search=...
- Several screens ask users/admins for raw IDs: GroupDetailPage pinned media, GroupDetailPage post media IDs, ModerationPage scope ID, ModerationReportRow group/user ban controls. Replace with reusable searchable entity pickers.
- UI is visually inconsistent because most pages define local Tailwind style dictionaries. Introduce shared Button, Field, Select, Dialog, DataTable, Badge, EmptyState, Tabs, and EntityPicker primitives.
- Admin/moderation screens need dense operational workflows rather than decorative card grids: search, pagination, filters, target previews, accessible confirmations, and summary counts.
- GroupsPage has client-side media/tag filters that inspect group text, not real metadata. Remove them or add backend-supported group tags/categories.
- Playwright smoke tests live under tests/e2e and currently cover route/search/picker flows with mocked network data rather than a seeded full-stack environment.
