6. Production bugs found and fixed
==================================

ReviewService:
- CreateReviewAsync allowed reviews for ratings whose media target was soft-deleted. It now rejects deleted media targets with KeyNotFoundException.
- CreateReviewAsync allowed reviews for ratings whose season target was soft-deleted. It now rejects deleted season targets with KeyNotFoundException.
- CreateReviewAsync allowed reviews for ratings whose episode target was soft-deleted. It now rejects deleted episode targets with KeyNotFoundException.
- UpdateReviewAsync allowed editing reviews after their target was deleted. It now rejects updates when the linked media/season/episode target is deleted.
- GetMediaReviewsAsync leaked reviews for deleted media. It now filters deleted media.
- GetUserReviewsAsync leaked reviews for deleted media. It now filters deleted media.
- Season/episode target validation also checks the owning TV series media is not deleted.

UserMediaStatusService:
- No production changes were required.
- Existing behavior already rejects nonexistent/deleted media on SetStatusAsync.
- Existing behavior already hides deleted media from GetUserStatusesAsync.
- Existing composite key and upsert path already prevent duplicate status rows for the same user/media.
