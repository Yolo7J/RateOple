4. OAuth redirect result handling
=================================

Added a dedicated frontend callback route:
- `/auth/callback`

Flow:
- Backend redirects back to `/auth/callback?returnUrl=...&externalLogin=success|failed[&error=...]`
- `ExternalLoginCallbackPage.jsx` refreshes frontend auth state through `AuthContext.refreshSession()`
- On success, the frontend redirects to the intended safe local route and removes callback params from the address bar
- On failure, the frontend redirects to login with a friendly message in navigation state instead of leaving OAuth params in the URL

Other auth redirect updates:
- `RequireAuth` now preserves `pathname + search + hash` through a safe `returnUrl` query param
- `useAuthSessionQuery` and `AuthContext` now share the same session refresh helper

Backend callback polish:
- `AuthController.GoogleLogin` now redirects back with `externalLogin=failed&error=not_configured` when Google auth is not configured instead of leaving the user on a backend `404` response
- `AuthController.GoogleCallback` uses the same fallback when Google auth is unavailable

5. Error handling changes
=========================

Improved auth error extraction in `frontend/src/features/auth/services/authService.js`.

Supported error shapes:
- string response bodies
- `error.response?.data?.message`
- `error.response?.data?.detail`
- `error.response?.data?.title`
- validation arrays with `description` values

User-facing Google auth failures now distinguish:
- generic Google sign-in failure
- backend-not-configured Google sign-in failure
