Shared design language changes
==============================

- Added semantic CSS variables for focus rings, radii, elevation, and status colors.
- Added reusable component classes for surfaces, cards, buttons, fields, badges, messages, empty states, tables, and skeletons.
- Normalized page and section title scale with `ui-page-title` and `ui-section-title`.
- Kept the visual direction calm and practical: mostly flat surfaces, restrained elevation, clear borders, limited accent use, no heavy decorative gradients.
- Improved table density and mobile behavior with a shared scroll-safe table wrapper.

Shared component changes
========================

- MediaCard is now a semantic link instead of a click-only article, improving keyboard access.
- CollectionCard, GroupCard, GroupPostCard, NotificationItem, AccountSection, and admin cards now use the shared card/button/badge language.
- AdminMediaPage now uses shared PageHeader, DataTable, Dialog, EmptyState, LoadingState, InlineMessage, and Button primitives.
- Moderation and audit pages now use shared headers, cards, loading/empty/error states, and badges.
- Auth forms now use shared Input, Button, and InlineMessage primitives.
- Discovery sections and media rows now use shared loading and empty states.

Global state pattern changes
============================

- Standardized common loading UI through LoadingState.
- Standardized no-data UI through EmptyState.
- Standardized error/info/success feedback through InlineMessage.
- Standardized confirmation modal behavior through Dialog.
- Added a mocked Playwright smoke test for shared UI rendering and horizontal overflow.

Accessibility/responsiveness improvements
=========================================

- Added consistent visible focus treatment for links, buttons, form controls, and focusable role elements.
- Improved keyboard navigation for shared media cards by using links.
- Improved EntityPicker ARIA roles for combobox/listbox/option behavior.
- Improved modal escape handling and background scroll locking in Dialog.
- Added header min-width and overflow protections.
- Added Playwright coverage for desktop and mobile discovery rendering without horizontal overflow.
- Disabled React Query devtools during e2e runs to avoid test-only viewport overflow.

Remaining UI polish gaps
========================

- Some deeper feature pages still have local one-off form/button styles, especially media add/edit/cart/season manager and detail-heavy group/collection pages.
- Some legacy confirmation flows still use `window.confirm` or `window.prompt` and should move to Dialog-driven flows.
- Tables/lists beyond AdminMediaPage can migrate to DataTable or shared list primitives.
- A full keyboard/focus trap implementation for Dialog would be stronger than the current lightweight Escape/scroll-lock behavior.
- Review/rating and watchlist surfaces can be further normalized with SectionCard, FormField, and Tabs.

Recommended next task
=====================

Run a focused page-by-page polish pass using the new primitives, starting with media add/edit/cart/season manager and then group/collection detail pages. Prioritize replacing one-off forms, confirmation prompts, and dense admin/moderation action clusters with the shared primitives introduced here.
