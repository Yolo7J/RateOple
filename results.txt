11. Production bugs found and fixed
===================================

MediaService:
- Missing genre IDs were silently ignored on create/update. They now throw KeyNotFoundException.
- Duplicate genre IDs could create nondeterministic relationship behavior. They now throw ArgumentException.
- Non-positive genre IDs now throw ArgumentException.
- TV-series episode upserts through MediaService used navigation collection adds that could surface EF concurrency/provider issues in tests. New episodes are now added through DbSet with explicit foreign keys.
- TV-series upsert payloads now reject non-positive and duplicate season/episode numbers.

TvSeriesService:
- Season number validation was mostly API-only. Service now rejects non-positive season numbers.
- Episode number validation was mostly API-only. Service now rejects non-positive episode numbers on add/update/delete paths.
- Duplicate episode numbers in a season payload are rejected before persistence.
- Adding an episode with a number already used by a soft-deleted episode now fails deterministically instead of depending on the database unique constraint.
- New episodes are added through DbSet with explicit foreign keys to avoid navigation tracking/provider-specific concurrency failures.

Test namespace issue fixed:
- API media tests use namespace RateOple.Api.Tests.MediaTests to avoid shadowing the Media entity type used by existing tests.
