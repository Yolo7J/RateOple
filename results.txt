2. Global search behavior implemented
=====================================

- Header search no longer logs to the console or leaves TODO behavior.
- Desktop and mobile header search submit to /media?search=<encoded query>.
- Search text is trimmed before navigation.
- Empty header search navigates to /media.
- The search icon is now a submit button, so Enter and click submission both work.
- Mobile search closes the search sheet after submit.
- Header search reflects the current /media search query when the URL contains search=...

3. Media results query-param behavior implemented
=================================================

- MediaListPage now treats URL query params as route state for search, type filters, genre filters, sort, and page.
- /media?search=inception calls the media list query with search=inception.
- The local media search input reflects the search query param.
- Clearing search removes search from the URL and refreshes results.
- Pagination updates page in the URL while preserving search/filter params.
- Empty search results show a query-specific empty state.
- Backend media search already supported title search and deleted media exclusion, so no backend rewrite was needed.

4. Group filter decision and changes
====================================

Decision: remove fake filters.

- Groups do not currently have real backend media-type or tag/category metadata.
- Removed the media type and tag dropdowns that previously filtered group name/description text.
- Removed the client-only sort control so the page does not imply unsupported server sorting.
- Kept backend-supported search.
- Added backend-supported visibility filtering.
- Added honest future-facing copy that discovery filters can return when groups support tags or categories.
