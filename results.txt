Permission behavior:
- SuperAdmin and Admin can see and action all reports.
- Moderator with Global assignment can see and action all reports.
- Moderator with scoped Group assignment sees/actions matching group-post and group-comment reports.
- Moderator with scoped Media assignment sees/actions matching media review/comment/post-media reports already represented by the report model.
- Moderator with no assignments gets an empty actionable queue, and the frontend shows: "You do not have any active moderation assignments."
- Normal User cannot access the global moderation queue.
- Group Owner, GroupAdmin, and GroupModerator remain group-domain roles only; they do not grant global moderation queue access.

Report actions implemented:
- Mark In Review.
- Resolve.
- Reject.
- Escalate to Admin.
- Optional moderation note/reason on status actions.
- Status changes write moderation audit logs, create existing report-status notifications for the reporter, and publish existing moderation realtime updates after commit.

Target types with hide/remove support:
- Group post comments represented as `ReportTargetType.Comment`, only when the target is a leaf comment in a group-post thread.
- Removal writes `ReportTargetRemoved` audit, resolves the report with a status audit entry, sends the existing `CommentRemoved` notification to the affected comment author where applicable, and publishes a report realtime update.

Target types still unsupported:
- User targets: no account removal/suspension action is exposed from the moderation queue.
- Review targets: existing review deletion is owner-only and not a safe moderator queue action.
- GroupPost targets: no safe moderator queue removal endpoint exists.
- Review comments: no safe queue removal path exists.
- Comments with replies: blocked to avoid unsafe hard-delete behavior under the current comment relationship.
- Collection, Group, and media catalog targets: not represented as safe moderation queue removals in v1; media catalog removal remains Admin/SuperAdmin territory.
