=================================

Covered rollback behavior:

- Existing media rating update rolls back rating value and aggregate when taste recalculation throws.

7. Moderation transaction tests added
=====================================

Added backend/RateOple.Core.Tests/Moderation/ModerationTransactionTests.cs.

Covered rollback behavior:
- Report status update rolls back when moderation audit logging throws.
- Report status update rolls back status and audit log when persisted notification creation throws.
- Moderator assignment rolls back when persisted notification creation throws.
- Moderator assignment removal rolls back when audit logging throws.

Production fix:
- ModerationService now wraps report status updates, assignment creation, assignment removal, audit logging, and persisted moderation notification creation in one EF transaction.
- Realtime moderation publishing now occurs after the database transaction commits.

Policy decision:
- Moderation audit logs are mandatory.
- Persisted moderation notifications are mandatory for tested moderation mutations.
- Realtime delivery is best-effort after commit.

8. Production bugs found and fixed
==================================

RatingService:
- Rating rows and media aggregates could commit before later interaction or taste failures.
- Rating deletes could commit before later taste failures.
- Fixed by adding transaction boundaries that are skipped when an outer transaction already exists.

InteractionService:
- MediaInteraction rows could commit before taste recalculation failed.
- This could leave false interaction signals affecting discovery/recommendations.
- Fixed by adding transaction boundaries that are skipped when an outer transaction already exists.

ModerationService:
- Report status and moderator assignment mutations could commit before audit or notification failures.
- Realtime notifications were invoked before an explicit transaction policy existed.
- Fixed by making audit/persisted notification side effects transactional and moving realtime publish calls after commit.

ReviewService:
- No new production fix required in this pass.
- Existing explicit transactions cover review writes and nested side-effect calls.

UserMediaStatusService:
- No new production fix required in this pass.
- Existing explicit transactions cover status writes and nested status-change interaction calls.

9. Transaction side-effect policy documented
============================================

Updated PROJECT_ARCHITECTURE.md.

Mandatory transactional side effects:
- rating aggregate refreshes
- rating-created and review-created interactions
- media-status-changed interactions
- synchronous taste recalculation from rating/review/interaction/status writes
- moderation audit logs
- persisted moderation notifications for report status and assignment changes

Best-effort side effects after commit:
- realtime SignalR delivery for moderation updates and notification fanout
- future non-persistent delivery integrations such as email or push notifications
