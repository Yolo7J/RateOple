- Preserved route paths, auth behavior, notifications, language switching, theme switching, search, and role-based staff links.

UX improvements:
- Rebuilt the desktop header with a sticky translucent surface, larger 44px action controls, clearer logo treatment, visible bell icon, readable unread badges, stronger search, and a polished account dropdown.
- Reworked the media dropdown with a single clear trigger, larger menu rows, icons, active state styling, click-outside closing, and Escape closing.
- Replaced the cramped mobile navigation with full-height app drawers for navigation and account, plus a separate search sheet with clear title, close button, and full-width search.
- Added drawer sections for Explore, Personal, Staff, Settings, and Account, preserving admin/moderation visibility rules.
- Redesigned the footer as a product footer with brand copy, route-based Explore/Community/Account links, auth-aware account links, and no fake contact information.
- Updated ThemeToggle and LanguageToggle to use larger visible controls with clearer state, icons, and readable EN/BG language labels.
- Removed the Google Fonts import and moved to a system font stack to avoid CSP font-loading issues.

Responsive behavior:
- Header uses the mobile/tablet shell below xl to avoid cramped intermediate layouts, then switches to full desktop navigation/search/actions at xl and above.
- Mobile controls use 44px minimum hit areas and the drawer width is constrained for 320px screens.
- Drawer bodies are scrollable for short-height devices.
- Global html/body/root overflow-x clipping was added to prevent horizontal scroll.
- Existing Playwright shared UI tests passed in desktop Chromium and mobile Chromium, including the horizontal-overflow guard.

Accessibility improvements:
- Icon-only controls have aria-labels.
- Menu, drawer, and dropdown triggers expose aria-expanded and aria-haspopup where relevant.
- Mobile panels use dialog semantics with aria-modal.
- Escape closes mobile panels and desktop dropdowns.
- Overlay click closes mobile panels.
- Focus ring styles remain visible through the global token system.

Checks run:
- npm run lint: passed.
- npm run build: passed. Build emitted existing Rollup annotation warnings from @microsoft/signalr comments.
- npm run test:e2e: passed, 2 tests.

Known limitations / manual inspection:
- Privacy and Terms are displayed in the footer bottom row as labels because no Privacy or Terms routes exist in frontend/src/app/router.jsx.
- Discovery/Home page content and MediaCard were intentionally not redesigned in this refactor.
