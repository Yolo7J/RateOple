8. Production bugs found and fixed
==================================

DiscoveryService:
- Trending had no deterministic tie-breaker. It now orders tied scores by title and id.
- Popular had no deterministic tie-breaker. It now orders tied popularity by title and id.
- Recommended had no deterministic tie-breaker. It now orders tied scores by title and id.
- Similar had no deterministic tie-breaker. It now orders tied scores by title and id.
- Discovery methods accepted non-positive limits directly. They now return an empty result for non-positive limits and cap positive limits at 100.
- Discovery DTO mapping omitted tags even though MediaListItemDto exposes Tags. Discovery now includes tags where loaded.
- Recommended excluded rated media but did not exclude media already marked Done. It now excludes completed media statuses.
- Recommended fallback for users with no taste data previously delegated to public popular results and could include completed media. It now applies the same per-user rated/completed exclusions before popularity fallback.
- Similar returned an empty list for missing or deleted source media, making not-found indistinguishable from "source has no genres." It now throws KeyNotFoundException.
- Similar ranking could allow a one-genre high-rating candidate to outrank a stronger multi-genre match. It now prioritizes genre overlap first, then rating-weighted score.

Deleted media protection:
- Trending excludes deleted media.
- Popular excludes deleted media.
- Recommended excludes deleted media.
- Similar excludes deleted source media and deleted candidates.
- Public discovery API responses are covered for deleted-media exclusion.
