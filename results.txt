3. InteractionService tests added
=================================

Added backend/RateOple.Core.Tests/Social/InteractionServiceTests.cs.

Covered behavior:
- Existing media can receive a MediaOpened interaction.
- Anonymous interaction behavior is explicit: InteractionService has no anonymous API and unknown users are rejected.
- Missing media cannot receive interactions.
- Deleted media cannot receive interactions.
- Stored interactions include user ID, media ID, interaction type, points, and timestamp.
- Repeated same-user same-media opens append new interaction rows.
- RatingCreated, ReviewCreated, and MediaStatusChanged interactions can coexist for the same user/media.
- Movies, books, and TV series can receive media-open interactions.
- Multiple or missing interaction targets are rejected.
- Season rating interactions store SeasonId and recalculate taste for the parent TV series.
- Episode review interactions store EpisodeId and recalculate taste for the parent TV series.

Current interaction boundary documented:
- Interactions are append-only.
- There is no anonymous MediaInteraction row shape because MediaInteraction.UserId is required.
- No clock abstraction exists; timestamp tests assert a bounded UTC range rather than an injected time.

4. UserTasteService tests added
===============================

Added backend/RateOple.Core.Tests/Social/UserTasteServiceTests.cs.

Covered behavior:
- Rating media with genres creates user genre scores.
- Multiple genres on one media split the rating value across genres.
- Updating a rating recalculates the score.
- Deleting a rating removes the stale score.
- Multiple ratings in the same genre aggregate deterministically.
- Repeated recalculation does not create duplicate UserGenreScore rows.
- Low ratings contribute less than high ratings under the current positive-point formula.
- Unrated media does not contribute.
- Deleted media does not contribute.
- Deleted media that previously had a stale UserGenreScore is removed on recalculation.
- Reviews and interactions contribute to taste under the current formula.
- Season ratings contribute to the parent TV series genres.
- Episode ratings contribute to the parent TV series genres.
- Deleted seasons do not contribute.
- Deleted episodes do not contribute.
- Deleted parent TV series prevent child ratings from contributing.
- Recalculation is scoped to one user and does not mutate another user's scores.
- Missing users are rejected.
- Deleted media contexts are rejected.

Current taste formula documented:
- Rating points equal the rating value.
- Review points are 6.
- Interaction points use the interaction's stored Points value.
- Points are distributed evenly across the resolved media's genres.
- Only positive aggregate scores are persisted.

6. API interaction smoke tests added
====================================

Added backend/RateOple.Api.Tests/Social/InteractionContractTests.cs.

Covered API behavior:
- Anonymous GET /api/media/{id} works but does not record a MediaOpened interaction.
- Authenticated GET /api/media/{id} records a MediaOpened interaction.
- Authenticated media-open recording on GET does not require CSRF because it is a safe-method read route.
- Deleted media returns 404 and does not record an interaction.
- Missing media returns 404 and does not record an interaction.
- Public trending reflects authenticated open interactions for non-deleted media only.

No standalone POST/PUT interaction endpoint exists, so no missing-CSRF mutation test was added for interactions.

7. Production bugs found and fixed
==================================

InteractionService:
- Unknown users previously reached database FK failure behavior. InteractionService now rejects unknown users with KeyNotFoundException before mutation.

UserTasteService:
- Direct ratings/reviews/interactions for soft-deleted media could still contribute to UserGenreScore during recalculation. Deleted media is now filtered during media resolution and genre distribution.
- Season ratings/interactions could resolve through a deleted parent TV series. Season resolution now requires an active parent media row.
- Episode ratings/interactions could resolve through a deleted episode season or deleted parent TV series. Episode resolution now requires active episode, season, and parent media rows.
- Stale UserGenreScore rows for deleted media are removed by recalculation when no active signals remain.

Deleted signal protection:
- Deleted media cannot receive new media-open interactions.
- Deleted media cannot contribute direct rating/review/interaction taste signals.
- Deleted seasons and episodes do not contribute child taste signals.
- Deleted parent TV series prevents season/episode child ratings from contributing.
- API media-open smoke tests verify deleted and missing media do not create interactions.
