3. Mocking strategy
===================

The E2E suite uses Playwright route mocks rather than backend fixture setup.

Mocks cover:
- /api/auth/me
- /api/auth/refresh
- /api/csrf
- /api/notifications
- media lookup
- public/admin user lookup
- group lookup
- collection lookup
- moderation scope lookup
- group detail, posts, pinned media, members, staff messages
- collection detail, child collection list, add-item mutation
- moderation reports, assignments, and group ban mutation

Mutation assertions inspect outgoing request bodies so tests verify that the UI selects humane labels while still sending internal IDs in API payloads.
